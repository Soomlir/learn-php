# Области видимости в PHP
В PHP области видимости (scopes) определяют, где можно обращаться к переменным и функциям. Это важный аспект, который влияет на то, как и где могут быть использованы различные элементы программы. В PHP существуют несколько типов областей видимости:

## 1. `Глобальная область видимости`
Переменные, объявленные вне всех функций и классов, находятся в глобальной области видимости. Они доступны во всей программе, за исключением функций и методов, где их нужно явно передавать или использовать.

Пример:
```php
$a = 10;  // Глобальная переменная

function test() {
    global $a;  // Доступ к глобальной переменной
    echo $a;    // Выведет 10
}

test();
```
Важное замечание: внутри функции или метода, переменные глобальной области не доступны напрямую, их нужно использовать через ключевое слово global.

## 2. `Локальная область видимости`
Локальные переменные существуют только внутри функции или метода. Они недоступны за пределами этих конструкций.

Пример:
```php
function test() {
    $b = 20;  // Локальная переменная
    echo $b;  // Выведет 20
}

test();
echo $b;  // Ошибка, переменная $b недоступна за пределами функции
```
Переменные, объявленные внутри функции, существуют только в пределах этой функции.

## 3. `Область видимости метода/свойства в классе`
В PHP классы могут использовать различные уровни видимости для своих методов и свойств. Это определяет, кто и как может использовать их.
- `public` — доступен везде (как изнутри класса, так и из-за его пределов).
- `protected` — доступен внутри класса и его дочерних классов.
- `private` — доступен только внутри класса, в котором он был объявлен.

Пример:
```php
class MyClass {
    public $publicVar = "Я публичная переменная";
    private $privateVar = "Я приватная переменная";
    protected $protectedVar = "Я защищенная переменная";

    public function test() {
        echo $this->publicVar;   // Доступ к публичному свойству
        echo $this->privateVar;   // Доступ к приватному свойству внутри класса
        echo $this->protectedVar; // Доступ к защищенному свойству внутри класса
    }
}

$obj = new MyClass();
$obj->test();  // Вызов метода, доступ к public, private и protected внутри класса

// echo $obj->privateVar;  // Ошибка: нельзя доступиться к private свойству вне класса
```

## 4. `Статическая область видимости`
Статические свойства и методы относятся к классу, а не к экземпляру класса. Такие элементы сохраняют свое значение между вызовами.

Пример:
```php
class MyClass {
    public static $counter = 0;

    public static function incrementCounter() {
        self::$counter++;  // Доступ к статическому свойству
    }
}

MyClass::incrementCounter();
echo MyClass::$counter;  // Выведет 1
```
Статические свойства и методы можно вызывать через класс без создания экземпляра этого класса.

## 5. `Область видимости замыканий (Closures)`
Замыкания — это анонимные функции, которые могут захватывать и использовать переменные из внешнего контекста.

Пример:
```php
$x = 10;

$closure = function() use ($x) {
    echo $x;  // Замыкание захватывает переменную $x
};

$closure();  // Выведет 10
```

## 6. `Область видимости через global и static`
- Ключевое слово `global` позволяет переменным из глобальной области видимости быть доступными в функции.
- Ключевое слово `static` позволяет сохранять значение переменной между вызовами функции, сохраняя её состояние в процессе выполнения программы.

Пример с `static`:
```php
function testStatic() {
    static $count = 0;
    $count++;
    echo $count;
}

testStatic();  // Выведет 1
testStatic();  // Выведет 2
```

Каждая область видимости играет свою роль в организации кода и помогает избежать ошибок при работе с переменными и методами, ограничивая доступ к данным, когда это необходимо.
