# Функция header в PHP
В PHP функция `header()` используется для отправки HTTP-заголовков клиенту до того, как начнется вывод контента на экран. Заголовки содержат важную информацию, которая передается браузеру или другому клиенту. Эта функция позволяет настроить различные аспекты HTTP-ответа, такие как редиректы, тип контента, кэширование и многое другое.

Синтаксис функции:
```php
header(string $header, bool $replace = true, int $response_code = 0): void
```
- `$header` — строка, содержащая сам заголовок (например, `Location: /new-page.php` или `Content-Type: text/html`).
- `$replace` — (опционально) флаг, который указывает, следует ли заменять текущий заголовок. Если `true`, заголовок будет заменен, если `false`, новый заголовок будет добавлен.
- `$response_code` — (опционально) HTTP-статус код. Если передан, будет отправлен HTTP-статус с этим кодом (например, 404 для страницы "не найдено").

Примеры использования:
## 1. Перенаправление (редирект):
Чтобы перенаправить пользователя на другой URL, используйте заголовок `Location`:
```php
header("Location: https://example.com");
exit; // Завершаем выполнение скрипта, чтобы не было вывода контента
```
После этого браузер перенаправит пользователя на указанный URL.

## 2. Установка типа контента:
Для указания типа контента (например, для JSON или изображений):
```php
header("Content-Type: application/json");
echo json_encode($data); // Выводим данные в формате JSON
```

## 3. Управление кэшированием:
Чтобы избежать кэширования страницы браузером, можно установить заголовки для кеширования:
```php
header("Cache-Control: no-cache, no-store, must-revalidate");
header("Pragma: no-cache");
header("Expires: 0");
```

## 4. Установка HTTP-статуса:
Вы можете установить любой HTTP-статус код. Например, если страница не найдена (404):
```php
header("HTTP/1.1 404 Not Found");
```

## 5. Отправка cookies:
Для отправки cookies можно использовать заголовок `Set-Cookie`:
```php
header("Set-Cookie: user=John; expires=Wed, 21 Oct 2025 07:28:00 GMT; path=/");
```

### Важные моменты:
1. Заголовки должны отправляться до вывода контента. Если на странице есть даже пробелы или переносы строк перед вызовом функции `header()`, возникнет ошибка. Поэтому всегда вызывайте `header()` до любого вывода (например, до `echo` или `print`).
2. Окончание выполнения скрипта. После отправки заголовков, особенно в случае редиректа, часто используется команда `exit`;, чтобы избежать дальнейшего выполнения скрипта.
3. Множественные заголовки. Если нужно отправить несколько заголовков, можно вызывать функцию `header()` несколько раз.
```php
header("Cache-Control: no-cache");
header("Location: /new-page.php");
```

Ошибки:
Если попытаться отправить заголовки после того, как уже был отправлен какой-либо вывод, PHP сгенерирует предупреждение:
```php
Warning: Cannot modify header information - headers already sent by (output started at /path/to/script.php:10)
```
Для предотвращения таких ошибок обычно используют буферизацию вывода (например, `ob_start()` в начале скрипта), которая позволяет избежать этой проблемы.
