# Позднее статическое связывание в PHP
Позднее статическое связывание (late static binding, LSB) — это механизм в PHP, который был введён в версии 5.3 и позволяет динамически вызывать статические методы в контексте классов, которые наследуют методы от родительского класса. Это особенно полезно, когда нужно обратиться к статическим методам или свойствам в контексте производного класса, а не родительского.

`Основная проблема, которую решает позднее статическое связывание`: До появления позднего статического связывания, когда вы использовали статические методы в классах и наследовали их, PHP всегда использовал имя родительского класса для разрешения статических методов. Однако иногда это было не то, что нужно, поскольку вы могли захотеть, чтобы использовалось имя производного класса.

## Как это работает?
В случае позднего статического связывания можно использовать специальный ключевое слово `static`, которое в контексте наследования будет указывать на имя класса, в котором был вызван метод, а не на класс, где он был определён.

Пример:
```php
class ParentClass {
    public static function whoAmI() {
        echo "Я — " . __CLASS__;
    }

    public static function test() {
        static::whoAmI();  // здесь используется static для позднего связывания
    }
}

class ChildClass extends ParentClass {
    public static function whoAmI() {
        echo "Я — " . __CLASS__;
    }
}

// Вызов метода в контексте дочернего класса
ChildClass::test();  // Выведет: Я — ChildClass
```

## Объяснение:
- В родительском классе `ParentClass` есть статический метод `test()`, который вызывает статический метод `whoAmI()`.
- В `whoAmI()` используется ключевое слово `__CLASS__`, которое указывает на класс, в контексте которого выполняется код.
- Когда метод `test()` вызывается через `ChildClass`, используется позднее статическое связывание. В результате `static::whoAmI()` вызывает метод `whoAmI()` класса `ChildClass`, а не родительского класса.

## Почему это полезно?
1. `Инкапсуляция`: Это позволяет дочерним классам изменять или переопределять статические методы, не нарушая логику работы родительского класса.
2. `Гибкость`: Мы можем динамически изменять поведение статических методов в разных контекстах, что дает большую гибкость при работе с наследованием.

## Пример с наследованием и переопределением:
```php
class Base {
    public static function hello() {
        echo "Hello from " . __CLASS__;
    }

    public static function callHello() {
        static::hello();
    }
}

class Derived extends Base {
    public static function hello() {
        echo "Hello from " . __CLASS__;
    }
}

// Вызов callHello из производного класса
Derived::callHello();  // Выведет: Hello from Derived
```

## Важные моменты:
1. `static::` — это ключевое слово, которое используется для позднего статического связывания. Оно позволяет обращаться к статическим методам и свойствам в контексте того класса, в котором метод был вызван, а не того, где он был определён.
2. `__CLASS__` — это магическая константа, которая возвращает имя класса, в котором находится код.

Позднее статическое связывание в PHP помогает создавать более гибкие и расширяемые классы, облегчая поддержку и расширение кода, особенно при работе с наследованием.
